# Configuration Guide

Welcome to the Instagram Reels Automation platform! All configuration is controlled using the `.env` file located in the root of your project.

To get started quickly, copy `.env.example` to `.env` and fill in your details.

---

## 1. Server Configuration
These settings control the web server that hosts your application.
- `PORT` *(Optional)*: The port the server runs on. (Default: `3000` or `8080`)
- `NODE_ENV`: Should be `production` when running on a live server, or `development` locally.
- `PUBLIC_URL` **(Required)**: The public internet URL where your app is hosted (e.g., `https://my-app.onrender.com`). **Instagram needs this to download your videos.**
- `API_KEY` **(Required)**: A strong, random password used to protect your webhook endpoints from hackers.
- `DRY_RUN` *(Optional)*: Set to `true` to test the app without actually posting to Instagram (simulates a successful upload).

---

## 2. Database
- `DATABASE_URL` **(Required)**: The connection string to your PostgreSQL database (e.g., Neon).

---

## 3. Storage (Local / Cloudflare R2)
Choose where to temporarily store videos before Instagram processes them.
- `STORAGE_PROVIDER`: Set to `local` to use the server's disk (default), or `r2` to upload to Cloudflare R2 (recommended for production).
- `R2_ACCOUNT_ID`: Your Cloudflare Account ID (Required if using `r2`).
- `R2_ACCESS_KEY_ID`: Cloudflare R2 Access Key (Required if using `r2`).
- `R2_SECRET_ACCESS_KEY`: Cloudflare R2 Secret Key (Required if using `r2`).
- `R2_BUCKET_NAME`: Cloudflare R2 Bucket Name (Required if using `r2`).

---

## 4. Meta / Instagram API
These link the app to your Meta Developer application.
- `GRAPH_API_TOKEN` **(Required)**: A long-lived Page Access Token generated from the Meta Developer Dashboard.
- `GRAPH_API_VERSION`: The version of the Meta API to use (e.g., `v19.0`).
- `TOKEN_EXPIRY_DATE`: The date your token expires (e.g., `2026-08-31`). The bot warns you 10 days before it expires.
- `FACEBOOK_PAGE_ID` *(Optional)*: Your connected Facebook Page ID for reference.

---

## 5. Google Drive API
These connect the app to Google Drive to download your videos.
- `GOOGLE_CLIENT_ID` **(Required)**: From Google Cloud Console -> Credentials.
- `GOOGLE_CLIENT_SECRET` **(Required)**: From Google Cloud Console -> Credentials.
- `GOOGLE_REFRESH_TOKEN` **(Required)**: Generated by running `npm run setup` locally.

---

## 6. Accounts Configuration
This connects your Google Drive folders to specific Instagram accounts and Telegram topics.
- `ACCOUNTS_CONFIG` **(Required)**: A JSON array containing the setup for each account you want to automate.

**Example ACCOUNTS_CONFIG Format:**
```json
[
  {
    "accountName": "My Account 1",
    "instagramAccountId": "178414XXXXXXX",
    "driveFolderId": "1A2b3C4d5E...",
    "driveUploadedFolderId": "5E4d3C2b1A...",
    "telegramThreadId": 1,
    "isNewAccount": true,
    "warmupStartDate": "2026-07-05",
    "proxyUrl": "socks5://user:pass@host:port"
  }
]
```

---

## 7. Upload Strategy & Limits
These settings control how fast and how often videos are uploaded.
- `POLLING_CRON`: How often to check Google Drive for new videos (e.g. `*/10 * * * *` means every 10 minutes).
- `UPLOAD_DELAY_SECONDS`: The strict wait time between consecutive uploads (e.g., `120` seconds).
- `DAILY_UPLOAD_LIMIT`: Maximum total videos to upload across all accounts per day.
- `TARGET_DAILY_LIMIT`: The target number of videos per account to upload per day.
- `POSTING_WINDOW_START`: The earliest time of day to post videos (e.g., `08:00`).
- `POSTING_WINDOW_END`: The latest time of day to post videos (e.g., `22:00`).
- `MAX_FILE_SIZE_BYTES`: Max video size allowed (e.g., `1073741824` for 1GB).
- `MAX_DURATION_SECONDS`: Max video length (e.g., `900` for 15 minutes).

---

## 8. Reliability & Health
These protect your Instagram accounts from bans and handle API errors.
- `ENABLE_ADAPTIVE_WARMUP`: Set to `true` to slowly increase uploads for brand new accounts over 14 days.
- `ENABLE_HEALTH_SCORING`: Set to `true` to auto-pause accounts that receive warnings from Instagram.
- `DEFAULT_COOLDOWN_HOURS`: How long an account should pause if its health score drops too low (e.g. `48` hours).
- `MAX_RETRY_ATTEMPTS`: How many times to retry an upload if Instagram's servers fail (e.g., `3`).
- `RETRY_BASE_DELAY_MS`: Initial wait time before the first retry (e.g., `5000` ms).
- `STATUS_POLL_INTERVAL_MS`: How often to check if Instagram finished processing the video.
- `STATUS_POLL_TIMEOUT_MS`: Max time to wait for Instagram to finish processing.

---

## 9. Content
- `CAPTION_FILE`: Path to a text file (e.g. `./caption.txt`) containing the caption used for every upload.
- `COVER_IMAGE` *(Optional)*: Path to an image (e.g. `./cover.jpg`) used as the cover for the reels.

---

## 10. Telegram Notifications
- `TELEGRAM_BOT_TOKEN`: The API token from BotFather.
- `TELEGRAM_CHAT_ID`: The ID of your Telegram Chat, Group, or Channel.
- `LARGE_QUEUE_WARNING_THRESHOLD`: Alert Telegram if more than this many videos are stuck in the queue.

---

## 11. Logging
- `LOG_LEVEL`: How much detail to log (`debug`, `info`, `warn`, `error`).
- `LOG_DIR`: Directory where log files are saved (e.g. `./logs`).
